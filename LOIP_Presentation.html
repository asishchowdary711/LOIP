01 — The Problem

The slowest, most fraud-exposed step in lending

The front door of every loan is manual: a team checks IDs against the application by hand, eyeballs pay stubs in a dozen formats, and computes affordability on a calculator. It's slow enough to kill conversion, error-prone, and the prime entry point for fraud.

🪪

Who is the borrower?

Extract from ID documents, cross-check against the application, flag tampering or identity mismatch (KYC).

💸

Can they afford it?

Reconstruct true income from messy income documents, detect manipulation, compute a defensible debt-to-income figure.

🎯

What's the decision?

Produce an overall onboarding risk score with an APPROVE / REVIEW / REJECT recommendation.

🔍

Why this decision?

Every number traced to its source document — explainable to both the bank officer and the applicant.

06 — Concept

What is a "wrapper"?

A wrapper is a class that hides one ML model or external service behind a uniform interface the pipeline can call without knowing the details. Every wrapper honours the same contract — and a single flag swaps fake data for real inference.

class SomeModelWrapper:
    def __init__(self, mock_mode=True):
        # mock_mode=True  → deterministic fake data (no GPU, no keys, instant)
        # mock_mode=False → load real model / call real API
        ...
    def predict(self, x):
        if self.mock_mode:
            return FakeResult(...)        # always works, reproducible
        return real_model.predict(x)      # real inference
🧩

Uniform interface

The pipeline calls every model the same way; internals differ, calling code never changes.

🔌

One-flag swap

mock → ollama → hf. Qwen auto-detects Ollama at startup and degrades gracefully.

💻

Runs anywhere

Full pipeline runs on a laptop with no GPU, no API keys, no external services.
